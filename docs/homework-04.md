
 بالإستناد إلى المعلومات الواردة [في الوظيفة رقم ثلاثة](homework-03.md). أنشئ تابع جديد اسمه `generateAccountStatement` وهو يقبل بارامتر واحد من النوع `string` يمثل رقم الحساب المراد توليد كشف الحساب له. ويرجع هذا التابع كائن من الشكل التالي:
 ```
 {
	 account_number: string,
	 statement: [
		{
			no: number,
			debit: number,
			credit: number,
			balance: number
		},
		...
	 ]
	 
 }
 ```

حيث `account_number` هو رقم الحساب المولّد له الكشف.
و `statement` هو كشف الحساب له.
و `no` هو رقم متسلسل للمدخلات في الكشف يبدأ من الرقم 1.
و `debit` هو قيمة المدين في حال كانت العملية الموافقة d
و `credit` هو قيمة الدائن في حال كانت العملية الموافقة c
و `balance` هو رصيد الحساب بعد تنفيذ هذا السطر.

مثال بالإستناد إلى [عينة البيانات الواردة في الوظيفة السابقة](homework-03.md#sample-input)، لدينا الحساب ذو الرقم 1000. جرت عليه عمليتين فقط:
```
'd|75 |1000|3400'
'c|640|1000|3200'
```

فهذا يعني أنّ خرج التابع `generateAccountStatement` في هذه الحالة هو:

```javascript
{
	account_number: '1000',
	statement: [
		{
			no: 1,
			debit: 75,
			credit: 0,
			balance: 75
		},
		{
			no: 2,
			debit: 0,
			credit: 640,
			balance: -565
		},
	]
}
```

ويكون رصيده النهائي هنا هو: 565-
