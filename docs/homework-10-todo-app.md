|-------|----------|

| تاريخ الإنشاء | ديسمبر 2025 |

## 1. المقدمة

### 1.1 الغرض من الوثيقة

تهدف هذه الوثيقة إلى تحديد المتطلبات الوظيفية وغير الوظيفية لتطبيق إدارة المهام بأسلوب كانبان (Kanban). توفر هذه الوثيقة مرجعاً شاملاً للمطورين والمصممين وفريق ضمان الجودة لفهم وتنفيذ النظام المطلوب.

### 1.2 نطاق المشروع

يتكون التطبيق من صفحة ويب واحدة تعرض لوحة كانبان لإدارة المهام. يمكن للمستخدم إنشاء قوائم متعددة، وكل قائمة يمكن أن تحتوي على عدد من المهام. يدعم التطبيق السحب والإفلات لإعادة ترتيب القوائم والمهام.

### 1.3 الجمهور المستهدف

أي مستخدم بحاجة لتطبيق إدارة مهام بسيط

---
## 2. نظرة عامة على النظام

### 2.1 وصف المنتج

تطبيق ويب بسيط لإدارة المهام يستخدم نمط لوحة كانبان. يعرض التطبيق مجموعة من القوائم (Lists) بشكل أفقي، حيث يمكن لكل قائمة أن تحتوي على مهام (Tasks) متعددة. يتيح للمستخدم تنظيم مهامه بطريقة بصرية وسهلة الاستخدام.

### 2.2 المكونات الرئيسية

| المكون | الوصف |
|--------|-------|
| لوحة كانبان (Board) | الحاوية الرئيسية التي تعرض جميع القوائم بشكل أفقي |
| القائمة (List) | القائمة تحتوي على مجموعة من المهام ذات الصلة مع عنوان قابل للتعديل |
| المهمة (Task/Todo) | بطاقة تحتوي على عنوان ووصف للمهمة مع إمكانية التعديل والحذف |
| شريط البحث (Search) | حقل للبحث في المهام |

## 3. المتطلبات الوظيفية

### 3.1 متطلبات إدارة القوائم (Lists)

#### FR-LIST-001: إنشاء قائمة جديدة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من إنشاء قائمة جديدة عبر زر "+ Add New List" |
| 2 | يجب أن تظهر القائمة الجديدة في نهاية القوائم الموجودة |
| 3 | يجب أن يكون للقائمة الجديدة اسم افتراضي قابل للتعديل |
| 4 | يجب عرض عداد يظهر عدد المهام في كل قائمة |

#### FR-LIST-002: تعديل اسم القائمة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من تعديل اسم القائمة بالنقر عليه |
| 2 | يجب حفظ الاسم الجديد تلقائياً عند الانتهاء من التعديل |
| 3 | يجب ألا يكون الاسم فارغاً |

#### FR-LIST-003: حذف قائمة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من حذف قائمة عبر قائمة الخيارات (⋯) |
| 2 | يجب عرض رسالة تأكيد قبل الحذف |
| 3 | يجب حذف جميع المهام الموجودة داخل القائمة عند حذفها |

#### FR-LIST-004: إعادة ترتيب القوائم

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من سحب وإفلات القائمة لتغيير موقعها |
| 2 | يجب حفظ الترتيب الجديد تلقائياً |

---

### 3.2 متطلبات إدارة المهام (Tasks)

#### FR-TASK-001: إنشاء مهمة جديدة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من إضافة مهمة جديدة عبر زر "+ Add Task" |
| 2 | يجب أن تحتوي المهمة على عنوان (إلزامي) |
| 3 | يجب أن تحتوي المهمة على وصف (اختياري) |
| 4 | يجب أن تظهر المهمة الجديدة في نهاية القائمة |

#### FR-TASK-002: تعديل المهمة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من تعديل عنوان المهمة |
| 2 | يجب أن يتمكن المستخدم من تعديل وصف المهمة |
| 3 | يجب عرض أيقونة تحرير (✏️) عند تمرير المؤشر على المهمة |

#### FR-TASK-003: حذف المهمة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من حذف المهمة |
| 2 | يجب عرض رسالة تأكيد قبل الحذف |

#### FR-TASK-004: تحديد المهمة كمكتملة

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من تحديد المهمة كمكتملة عبر علامة (✓) |
| 2 | يجب عرض حالة "Completed" بشكل مرئي على المهمة المكتملة |

#### FR-TASK-005: إعادة ترتيب المهام

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من سحب وإفلات المهمة لتغيير ترتيبها داخل نفس القائمة |
| 2 | يجب حفظ الترتيب الجديد تلقائياً |

#### FR-TASK-006: نقل المهمة بين القوائم

| الرقم | المتطلب |
|-------|---------|
| 1 | يجب أن يتمكن المستخدم من سحب المهمة وإفلاتها في قائمة أخرى |
| 2 | يجب عرض منطقة إفلات "Drop task here" عند السحب |

## 5. نموذج البيانات

### 5.1 هيكل القائمة (List)

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | String (UUID) | معرف فريد للقائمة |
| name | String | اسم القائمة (مثال: "Tasks to Do") |
| order | Number | ترتيب القائمة في اللوحة |
| color | String | لون مؤشر حالة القائمة |
| tasks | Array\<Task\> | مصفوفة المهام الموجودة في القائمة |

### 5.2 هيكل المهمة (Task)

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | String (UUID) | معرف فريد للمهمة |
| title | String | عنوان المهمة (إلزامي) |
| description | String | وصف المهمة (اختياري) |
| isCompleted | Boolean | حالة اكتمال المهمة |
| order | Number | ترتيب المهمة داخل القائمة |
| listId | String | معرف القائمة التي تنتمي إليها المهمة |

## 6. واجهة المستخدم

### 6.1 مكونات الواجهة

بناءً على التصميم المرجعي، تتكون الواجهة من:

#### شريط العنوان (Header)

- شعار التطبيق واسم المشروع "Project Alpha"
- حقل البحث عن المهام
- زر "+ New Board" لإنشاء لوحة جديدة
- صورة المستخدم الشخصية

#### لوحة كانبان (Kanban Board)

- قوائم مرتبة أفقياً (Tasks to Do, In Progress, Completed Work)
- كل قائمة تحتوي على: عنوان مع مؤشر لوني، عداد المهام، قائمة خيارات
- منطقة إفلات المهام "Drop task here"
- زر "+ Add Task" في أسفل كل قائمة

#### بطاقة المهمة (Task Card)

- عنوان المهمة (بخط عريض)
- وصف المهمة (بخط أصغر)
- أيقونة التحرير (تظهر عند تمرير المؤشر)
- علامة الإكمال للمهام المكتملة

## 7. المواصفات التقنية

### 7.1 التقنيات المقترحة

| التقنية | الاستخدام |
|---------|----------|
| HTML5 / CSS3 | هيكل الصفحة والتنسيق |
| JavaScript (ES6+) | المنطق البرمجي والتفاعلية |
| React.js / Vue.js | إطار العمل المقترح لبناء الواجهة (اختياري) |
| CSS Grid / Flexbox | تخطيط لوحة كانبان |
| HTML Drag & Drop API | وظيفة السحب والإفلات |
| Local Storage API | تخزين البيانات محلياً |

### 7.2 متطلبات المتصفح

يجب أن يعمل التطبيق على أحدث إصدارات المتصفحات التالية:

- Google Chrome
- Mozilla Firefox
- Microsoft Edge
- Safari
